# OKR ダッシュボード - リデザインブリーフ

Claude Design (https://claude.ai/design) に渡すための、コードベース理解のための仕様書。
このファイルをまるごと Claude Design のプロンプトに貼り付け、各画面のスクショを添えてください。

## 1. 製品概要

「**タスク中心の業務運用 OS**」(キャッチコピー: 社員が今日何をすれば良いか迷わない)

- Notion / Asana のような点のツールではなく、1 日の業務フロー全体を線でつなぐツール
- 朝会 / 週次 MTG / OKR / タスク / 共有・確認 / Google 連携を統合
- ターゲット: 30〜200 人の成長企業 (B2B、月額 ¥1,980/人 + 初期導入費)

## 2. 技術スタック (= デザイン制約)

- Next.js 14 (App Router) / React 18
- Supabase (DB + Auth)
- 既存のデザイントークン: `lib/themeTokens.js` (light/dark の 2 テーマ)
  - 色: `T.bg / T.bgCard / T.border / T.text / T.textSub / T.textMuted / T.accent / T.success / T.warn / T.danger`
  - 角丸: `RADIUS.{xs,sm,md,lg,xl,2xl,pill}` (= 6 / 8 / 10 / 14 / 18 / 22 / 99)
  - 余白: `SPACING.{xs,sm,md,lg,xl,2xl,3xl}` (= 4 / 8 / 12 / 16 / 20 / 24 / 32)
  - 影: `SHADOWS.{xs,sm,md,lg,xl,hover,hero}`
  - タイポ: `TYPO.{largeTitle,title1,title2,title3,headline,body,callout,subhead,footnote,caption}`
- スタイルファクトリ: `lib/iosStyles.js` (cardStyle / btnPrimary / pillStyle 等)

## 3. デザインの方向性 (推奨)

- **Linear / Notion 風のミニマル・洗練** 寄り
- ただし業務密度が高いので Stripe / Vercel ダッシュボード風の情報量も両立させたい
- 日本語 UI 最適化 (Noto Sans JP / Hiragino Kaku Gothic ProN)

## 5. 全画面リスト

### 5.1 ポータル (= ホーム) — `PortalPage.jsx`

URL: `/{orgSlug}?page=portal`

**機能**: 内蔵ダッシュボード (ワークスペース) + 外部サービスダッシュボード集 (組織別カスタマイズリンク) のホーム画面

**主要 UI**:
- 大きい挨拶バナー (時間帯で「おはようございます」「こんにちは」)
- 検索バー (ダッシュボード横断検索)
- カードグリッド: 主要 / 事業ダッシュボード / 管理ツール / カスタムリンク
- 各カードに icon + title + description + 「アプリ内で開く →」or 「外部リンク」
- ユーザーが自由にカスタムリンクを追加可能 (localStorage)

### 5.2 ワークスペース (= 旧マイページ) — `MyPageShell.jsx`

URL: `/{orgSlug}?page=mycoach`

**機能**: 個人ダッシュボード + 全体サマリーモード切替。中央に大きなタブが並ぶ。

**主要 UI**:
- 左: 全体サマリー / 個別メンバー切替リスト
- 上タブ (個人モード): ダッシュボード / 経営戦略 / チームサマリー / 共有・確認 / タスク / メール / マイルストーン / OKR / カレンダー / ドライブ / MyCOO / 振り返り / 連携
- 中央: 選択されたタブのコンテンツ

### 5.3 ダッシュボードタブ (個人モード) — `MyPageShell.jsx` の dashboard セクション

**機能**: 「始業ゲート」(平日始業時の強制画面)、「今日やること」、「今週やること」、「直近のリマインダー」を集約

**主要 UI**:
- 平日始業時の `MorningGate` 画面 (フルスクリーンモーダル) - 「今日やること」を 1 件以上登録するまで先に進めない
- 始業後: 上部に work_log status (始業時刻 / 終業時刻) + 今日やること / 今週やること / 成果リスト
- 「終業する」ボタンで KPT (Keep/Problem/Try) 入力モーダル

### 5.4 全体サマリーモード — `CompanyDashboardSummary.jsx`

URL: `/{orgSlug}?page=mycoach` (全体サマリーモード)

**機能**: 全社向けダッシュボード

**主要 UI**:
- 全社通期 KR カード (4 件横並び、進捗率 + 担当)
- 注力マイルストーンカード (4 件、1 行)
- サブタブ: 概要 / 週間ランキング / マイルストーン
- 概要: 今すぐ目を向けるべきこと (アラート) / 今日の全社状況 / 前進した KR (3 列)
- 週間ランキング: 有言実行王 / タスク完了王 / 振り返り王 / 実践王 (4 列、Top 3 ずつ)

### 5.5 チームサマリー — `MyPageShell.jsx` の team_summary タブ

**機能**: チーム単位の Good / More / Focus を AI で集約

**主要 UI**:
- 上部に部署タブ (横並びボタン)
- 部署選択後にチームプルダウン
- 3 カラム: Good / More / Focus の textarea
- 「🤖 AIで生成」ボタン (= 過去のチームメンバーの KR/KA レビューから AI 自動生成)

### 5.6 OKR 年間ビュー — `AnnualView.jsx`

URL: `/{orgSlug}?page=okr` の「📊 年間」サブタブ

**機能**: 組織階層 + 親子 OKR + 通期/Q1/Q2/Q3/Q4 切替

**主要 UI**:
- 左に組織階層ツリー (経営 → 事業部 → チーム)
- 中央に Objective カード + Key Result カード + Key Action リスト
- Objective ごとに横スクロール (= 階層 ×期間 のグリッド)
- 進捗バー + 星 5 段階評価 (★0〜★5: 達成率 60% 未満〜150% 超)

### 5.7 OKR 個人ビュー — `MyOKRPage.jsx`

URL: `/{orgSlug}?page=okr` の「👤 個人」サブタブ

**機能**: 自分が担当の OKR / KR / KA を 1 画面で

**主要 UI**:
- 上部に「通期 / Q1 / Q2 / Q3 / Q4」サブ切替
- 自分の Objective → 配下 KR → 各 KR に紐付く KA リスト
- 各 KA にステータス (focus / good / more / done) アイコン + 担当者アバター

### 5.8 OKR 週次ビュー — `WeeklyMTGPage.jsx` の一覧モード (forceMode="list")

URL: `/{orgSlug}?page=okr` の「📅 週次」サブタブ

**機能**: 全 KR/KA を週単位の俯瞰ビュー

**主要 UI**:
- 会議切替プルダウン (= 部署別フィルタ)
- 3 ペイン: 通期 KR / Q期 KR / KA
- 各 KA にステータスバッジ + 担当アバター + KPT 入力欄

### 5.9 週次MTG (ファシリモード) — `WeeklyMTGFacilitation.jsx`

URL: `/{orgSlug}?page=weekly`

**機能**: 進行ガイド付きステップ式会議進行

**主要 UI**:
- 上部に会議選択 + ファシリモード/一覧モード切替
- ステップ式進行 (= 部署選択 → KA レビュー → KR レビュー → 課題・依頼事項 → 確認事項 → ネクストアクション)
- 進捗ステッパー (上部に各ステップのチップ)
- 各ステップに固有 UI (KA レビュー = 担当者順送り、KR レビュー = KPT 記入、等)

### 5.10 朝会 — `MorningMeetingPage.jsx`

URL: `/{orgSlug}?page=morning`

**機能**: 朝会の進行管理 (= ファシリテーター + 個別報告 → 確認事項 → ネクストアクション)

**主要 UI**:
- 上部: 「個別報告 → 確認事項 → ネクストアクション → 終了」のステップバー
- 中央: 現在のステップ
  - 個別報告: メンバー順送り (= 各メンバーの昨日 KPT + 今日のタスク を 1 人ずつ)
  - 確認事項: 双方向の確認/返信スレッド一覧
  - ネクストアクション: 会議で出たタスクを ka_tasks に登録

### 5.11 組織 — `OrgPage.jsx`

URL: `/{orgSlug}?page=orgjd`

**機能**: 組織管理 (階層 / メンバー / 職務記述書 / 業務一覧 / マニュアル)

**主要 UI**:
- 左: 階層ツリー (経営 → 事業部 → チーム)
- 中央: 選択した部署/チームのメンバー一覧 + 業務一覧
- 各メンバーに JD (Job Description) 編集モーダル

### 5.12 組織設定モーダル — `OrgSettingsPanel.jsx` + `OrgMeetingsSection.jsx`

トリガー: `OrgIconBar.jsx` の「⚙ 設定」ボタン

**機能**: 組織のライセンス / メンバー管理 / Notion 連携 / 会議カスタマイズ

**主要 UI**:
- myAI ライセンス状態 + 「myAI で購入」リンク
- 「+ メンバーを招待」ボタン → モーダル (Resend 経由でメール送信)
- Notion 連携設定 (DB ID マッピング)
- **会議設定 (Phase 5e)**:
  - 会議一覧 (各会議: アイコン / 名前 / モジュール構成タグ)
  - 「👁 プレビュー」「✏️ 編集」「🗑️ アーカイブ」ボタン
  - 「+ 新規会議を追加」ボタン → MeetingEditModal (= タイトル / アイコン / 色 / 曜日 / モジュール並び替え D&D)
- メンバー一覧 (各メンバー: 名前 / email / 部署 / ロール選択)

## 7. 共通 UI 要素 (デザイントークン拡張時に考慮)

- **アバター**: 名前頭文字 + slug ベースの色 (10 色パレット)
- **進捗バー**: 0-100% の横バー、達成率に応じて色変化 (赤 < 60 / 黄 60-100 / 緑 100+)
- **ステータスバッジ**: focus (🎯) / good (✅) / more (🔺) / done (✓) / normal (—)
- **ピル (Pill)**: 部署バッジ等。`pillStyle({ color, size, solid })` で生成
- **カード**: `cardStyle({ T, accent, padding })` で生成 (全画面で統一)
- **ボタン**: `btnPrimary` / `btnSecondary` / `btnGhost` / `btnDanger`
